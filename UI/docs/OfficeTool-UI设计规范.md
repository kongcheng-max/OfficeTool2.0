## 1. 设计气质与原则

**气质关键词**：清爽 · 专业 · 友好 · 有呼吸感

对标飞书（Feishu/Lark）的企业 SaaS 视觉：浅色为主、大留白、卡片化承载信息、柔和圆角与轻阴影，避免「深色后台管理系统」的沉重感。

### 设计原则

| 原则 | 含义 |
|---|---|
| **清晰优先** | 信息层次靠字号/字重/间距/留白建立，而非靠边框和分隔线堆砌。 |
| **一处定义** | 所有颜色、间距、圆角走语义化 token，禁止在页面里硬编码色值。 |
| **克制用色** | 大面积中性色（白/浅灰），主色只用于「可交互 / 需强调」元素。 |
| **8pt 网格** | 所有间距、尺寸对齐 4/8 的倍数，保证节奏统一。 |
| **双主题等价** | 亮色与暗色是同一套语义 token 的两组取值，任何新组件必须两套都成立。 |
| **状态可读** | 成功/警告/危险/信息四类语义色贯穿全局，含义固定不复用。 |

## 2. 品牌与 Logo

### 2.1 Logo 概念

一个**圆角方形徽标（App Mark）**，表达「文档解析 → 知识图谱」这条产品主线：

```
   ┌─────────────┐
   │  ┌────────┐╲│   ← 飞书蓝底 (#3370FF) 圆角方形 (12px radius)
   │  │        │ ╲   ← 白色「文档」带右上折角
   │  │        │  │
   │  │   ●    │  │   ← 文档下方延伸出 3 个相连节点
   │  │  ╱ ╲   │  │      (● 主节点 + 2 个子节点，白色/浅色连线)
   │  │ ●   ●  │  │      呼应「解析出的知识图谱」
   │  └────────┘  │
   └─────────────┘
```

**语义**：折角文档 = 文档解析；下方节点网络 = 知识图谱与实体关系。两者合一即「把文档解析成知识」。

### 2.2 组合方式

- **完整标**：徽标 + 文字标 `OfficeTool`，下方可加中文副标「企业文档智能解析」。
- **导航图标栏**：仅用徽标图形（正方形，圆角 12px，约 32×32）。
- **登录页 / 品牌区**：完整标（横向或竖向）。

### 2.3 变体与规范

| 变体 | 用途 |
|---|---|
| 主版（蓝底白图形） | 亮色背景、默认 |
| 反白版（白底蓝图形 / 透明底） | 深色背景、暗色主题顶栏 |
| 单色版（纯 `#1F2329` 或纯白） | 打印、水印、置灰场景 |

- **最小尺寸**：徽标不小于 24×24px；带文字标时不小于 96px 宽。
- **安全边距**：徽标四周留白 ≥ 徽标高度的 25%。
- **禁止**：拉伸变形、更换品牌色、加投影描边、旋转。

## 3. 色彩系统（亮/暗双主题）

所有颜色以**语义 token** 命名。页面只引用语义名，不直接写十六进制。亮/暗为同一 token 的两组取值。

### 3.1 品牌 & 功能色

| Token | 语义 | 亮色 Light | 暗色 Dark |
|---|---|---|---|
| `--color-primary` | 主色 | `#3370FF` | `#4E83FD` |
| `--color-primary-hover` | 主色 hover | `#245BDB` | `#6A97FE` |
| `--color-primary-active` | 主色 按下 | `#1C4AC9` | `#3D71E8` |
| `--color-primary-bg` | 主色浅底 | `#EBF1FF` | `rgba(78,131,253,.16)` |
| `--color-success` | 成功 | `#34C724` | `#3BD638` |
| `--color-success-bg` | 成功浅底 | `#E8FAE6` | `rgba(59,214,56,.16)` |
| `--color-warning` | 警告 | `#FF8800` | `#FF9F2E` |
| `--color-warning-bg` | 警告浅底 | `#FFF3E6` | `rgba(255,159,46,.16)` |
| `--color-danger` | 危险 | `#F54A45` | `#F76965` |
| `--color-danger-bg` | 危险浅底 | `#FDECEC` | `rgba(247,105,101,.16)` |
| `--color-info` | 信息 | `#3370FF` | `#4E83FD` |

### 3.2 中性色（文字 / 背景 / 边框）

| Token | 语义 | 亮色 Light | 暗色 Dark |
|---|---|---|---|

| `--bg-body` | 页面底 | `#F5F6F7` | `#17181A` |
| `--bg-container` | 卡片/容器 | `#FFFFFF` | `#242527` |
| `--bg-nav` | 侧边/顶栏 | `#FFFFFF` | `#1F2023` |
| `--bg-hover` | 悬浮填充 | `#F2F3F5` | `rgba(255,255,255,.08)` |

| `--border` | 边框 | `#DEE0E3` | `#3A3D42` |

原配色偏鲜艳跳跃，重新校准为同一亮度/饱和层级，暗色下降低亮度提高可读性。

| 实体 | 亮色 | 暗色 |
|---|---|---|
| PERSON 人物 | `#3370FF` | `#4E83FD` |
| ORG 组织 | `#14B8A6` | `#2DD4BF` |
| DATE 日期 | `#F59E0B` | `#FBBF24` |
| MONEY 金额 | `#34C724` | `#3BD638` |
| LOCATION 地点 | `#8B5CF6` | `#A78BFA` |

### 3.4 检索来源标签色

| 来源 | 语义 | 色 |
|---|---|---|
| VECTOR 向量 | 主色系 | `--color-primary` |
| BM25 关键词 | 成功系 | `--color-success` |
| KG 图谱 | 警告系 | `--color-warning` |

### 3.5 置信度 / 状态映射

| 状态 | 色 |
|---|---|
| 高置信 / ready 就绪 / 成功 | `--color-success` |
| 中置信 / processing 处理中 / 警告 | `--color-warning` |
| 低置信 / failed 失败 / 危险 | `--color-danger` |
| uploaded 已上传 / 中性 | `--text-secondary` |

## 4. 字体系统

### 4.1 字体族

```
font-family:
  "Inter",                          /* 拉丁/数字 */
  "PingFang SC",                    /* macOS 中文 */
  "Microsoft YaHei",                /* Windows 中文 */
  "Noto Sans SC",                   /* 兜底中文 */
  -apple-system, BlinkMacSystemFont, "Segoe UI", Roboto, sans-serif;
```

等宽（代码/ID/数字表格）：`"JetBrains Mono", "SFMono-Regular", Consolas, monospace`。

### 4.2 字号阶梯

| Token | 字号 | 行高 | 字重 | 用途 |
|---|---|---|---|---|
| `--font-display` | 30px | 40px | 600 | 页面大标题（仪表盘欢迎语） |
| `--font-h1` | 24px | 32px | 600 | 页面主标题 |
| `--font-h2` | 20px | 28px | 600 | 卡片/区块标题 |
| `--font-h3` | 16px | 24px | 600 | 小节标题 |
| `--font-body` | 14px | 22px | 400 | **正文默认** |
| `--font-body-strong` | 14px | 22px | 500 | 正文强调 |
| `--font-caption` | 12px | 18px | 400 | 辅助/标签/时间戳 |

- 数字（统计卡 KPI 值）可用 20–30px、字重 600、`font-variant-numeric: tabular-nums` 保证对齐。

## 5. 间距 / 圆角 / 阴影 / 动效

### 5.1 间距（8pt 网格）

| Token | 值 | 典型用途 |
|---|---|---|
| `--space-1` | 4px | 图标与文字间隙 |
| `--space-2` | 8px | 紧凑内边距、标签内距 |
| `--space-3` | 12px | 表单项间距 |
| `--space-4` | 16px | **卡片内边距、常规间隔** |
| `--space-6` | 24px | 区块间距、页面内边距 |
| `--space-8` | 32px | 大区块分隔 |

### 5.2 圆角

| Token | 值 | 用途 |
|---|---|---|
| `--radius-sm` | 4px | 标签、小按钮、输入框内元素 |
| `--radius-md` | 6px | **按钮、输入框（默认）** |
| `--radius-lg` | 8px | 卡片 |
| `--radius-xl` | 12px | 大容器、模态、Logo 徽标 |
| `--radius-full` | 999px | 头像、胶囊标签、开关 |

### 5.3 阴影（柔和三档）

| Token | 值（亮色） | 用途 |
|---|---|---|
| `--shadow-sm` | `0 1px 3px rgba(31,35,41,.08)` | 卡片 hover 抬升 |
| `--shadow-md` | `0 4px 16px rgba(31,35,41,.10)` | 下拉、气泡、浮层 |
| `--shadow-lg` | `0 8px 32px rgba(31,35,41,.16)` | 模态、抽屉 |

暗色下阴影改用更深的黑（`rgba(0,0,0,.32~.48)`）并配合边框区分层级。

### 5.4 动效

| 场景 | 时长 | 缓动 |
|---|---|---|
| hover / 颜色变化 | 120ms | `ease-out` |
| 展开/折叠 / 抽屉 | 200ms | `cubic-bezier(.4,0,.2,1)` |
| 模态出现 | 240ms | `cubic-bezier(.34,1.56,.64,1)`（轻微回弹） |

- 尊重 `prefers-reduced-motion`，用户开启时关闭非必要动画。

## 6. 导航骨架

**飞书式双栏**：最左窄图标栏（全局模块）+ 第二级列表栏（上下文列表）+ 顶栏 + 内容区。侧边栏整体**浅色**，告别原深藏青 `#001529`。

```
┌────┬──────────────┬────────────────────────────────────┐
│    │              │  顶栏: 面包屑    🔍搜索  ☾主题  👤用户 │
│ 图 │  二级列表      ├────────────────────────────────────┤
│ 标 │  (随模块变化)  │                                    │
│ 栏 │              │                                    │
│    │  · 知识库列表  │          内容区（卡片化）            │
│~64 │  · 会话历史    │                                    │
│ px │  · 文档树      │                                    │
│    │  (可折叠 ◧)   │                                    │
│ 👤 │              │                                    │
└────┴──────────────┴────────────────────────────────────┘
 240px 二级列表
```

### 6.1 图标栏（~64px，固定）

- 顶部：Logo 徽标。
- 主导航图标（含文字或 tooltip）：**仪表盘 / 知识库 / 问答 / 图谱 / 管理**。
- 底部：主题切换（☀/☾）、用户头像。
- 选中态：图标高亮为 `--color-primary`，左侧 3px 主色条 + `--bg-active` 底。

### 6.2 二级列表栏（~240px，可折叠）

内容随当前模块变化：

| 模块 | 二级栏内容 |
|---|---|
| 知识库 | 知识库列表（搜索 + 「+新建」） |
| 问答 | 会话历史（按时间分组，「+新会话」） |
| 图谱 | 实体分类/图例 + 实体搜索 |
| 文档（进入某 KB） | 文档树 / 标签筛选 |

- 折叠后仅剩图标栏，内容区变宽。
- 列表项：hover `--bg-hover`，选中 `--bg-active` + 主色文字。

### 6.3 顶栏（~56px）

- 左：面包屑（当前位置路径）。
- 中/右：全局搜索框、亮/暗主题切换、通知（可选）、用户下拉（资料/设置/登出）。

### 6.4 响应式

- `≥1200px`：三栏全展开。
- `768–1200px`：二级栏默认折叠，可临时展开为浮层。
- `<768px`：图标栏收为底部 Tab Bar 或汉堡抽屉；二级栏与内容区堆叠。

## 7. 逐页版式设计

### 7.1 登录 / 注册

替换现有紫色渐变，改为飞书蓝品牌氛围。左品牌展示区 + 右表单卡。

## 8. 组件规范

各组件给出状态与 token 映射。默认圆角 `--radius-md`，过渡 120ms。

### 8.1 按钮

| 类型 | 默认 | hover | active | disabled |
|---|---|---|---|---|
| 主按钮 Primary | 底 `--color-primary` / 白字 | 底 `--color-primary-hover` | 底 `--color-primary-active` | 底 `--text-placeholder` / 白字 |
| 次按钮 Default | 底 `--bg-container` / 边框 `--border` / 文字 `--text-primary` | 边框+文字变 `--color-primary` | 底 `--bg-active` | 文字 `--text-placeholder` |
| 文字按钮 Text | 透明 / 文字 `--text-primary` | 底 `--bg-hover` | 底 `--bg-active` | 文字 `--text-placeholder` |
| 危险按钮 Danger | 底/文字 `--color-danger` | 加深 | 更深 | 置灰 |

- 尺寸：large 40px / **default 32px** / small 24px 高度。

### 8.2 输入框 / 表单

- 默认：底 `--bg-container`、边框 `--border`、圆角 `--radius-md`、内距 8×12。
- focus：边框 `--color-primary` + 2px `--color-primary-bg` 外发光。
- error：边框 `--color-danger` + 底部错误文案（`--color-danger`）。
- 占位文字 `--text-placeholder`。

### 8.3 卡片

- 底 `--bg-container`、圆角 `--radius-lg`、内距 `--space-4`、无默认阴影或极轻边框。
- hover（可点击卡片）：`--shadow-sm` 抬升 + `translateY(-2px)`。

### 8.4 标签 Tag / 胶囊

- 圆角 `--radius-sm`（方形标签）或 `--radius-full`（胶囊）。
- 语义色标签用对应 `*-bg` 底 + 主色文字（如成功：`--color-success-bg` 底 + `--color-success` 字）。

### 8.5 状态徽标（文档状态）

| 状态 | 呈现 |
|---|---|
| ready 就绪 | ● `--color-success` + 「就绪」 |
| processing 处理中 | ◐ `--color-warning` + 「处理中」（可带旋转/进度） |
| uploaded 待解析 | ○ `--text-secondary` + 「待解析」 |
| failed 失败 | ✕ `--color-danger` + 「失败」 |

### 8.6 表格

- 表头：底 `--bg-hover`、文字 `--text-secondary`、字重 500。
- 行：hover `--bg-hover`；斑马纹可选（暗色慎用）。
- 分隔用 `--divider`，避免重边框。

### 8.7 模态 / 抽屉

- 遮罩：`rgba(0,0,0,.45)`。
- 容器：`--bg-container`、圆角 `--radius-xl`（模态）、`--shadow-lg`。
- 抽屉从右滑入，宽度 400–560px。

### 8.8 消息气泡（见 7.5）
### 8.9 来源引用卡（见 7.5）

### 8.10 空状态 / 加载 / 头像

- 空状态：居中插画/图标（`--text-placeholder`）+ 说明 + 主要操作按钮。
- 加载：骨架屏（卡片/表格）优先于单个 spinner；主色 spinner 用于按钮内联。
- 头像：圆形 `--radius-full`；无图时用首字母 + 语义色底。

## 9. 实施映射与落地指引

> 本文档为设计交付物，不改动现有代码。以下为后续实现阶段的落地建议。

### 9.1 两个中心杠杆点

1. **`app/frontend/src/App.tsx`** —— AntD `ConfigProvider` 的 `theme.token`（当前仅 `colorPrimary: '#1677FF'` + `borderRadius: 6`）。落地时扩展为完整 token 映射，并接入 `theme.algorithm`（`defaultAlgorithm` / `darkAlgorithm`）实现亮暗切换。
2. **`app/frontend/src/index.css`** —— 全局样式。落地时在此定义 CSS 变量（`:root` 亮色 + `[data-theme="dark"]` 暗色），字体族在此声明。

### 9.2 建议新增

- `app/frontend/src/theme/tokens.ts` —— 集中导出本规范第 3–5 章的语义 token（供 AntD ConfigProvider 与 inline 样式共同引用）。
- `app/frontend/src/theme/ThemeProvider.tsx` —— 包裹 ConfigProvider，读取/持久化用户主题偏好（localStorage），提供切换。
- 主题切换开关放入顶栏与图标栏底部（见 6.1 / 6.3）。

### 9.3 inline 硬编码色值热点清单（落地时替换）

| 位置 | 现硬编码 | 替换为 token |
|---|---|---|
| `components/AppLayout.tsx` | 侧边栏 `#001529` | `--bg-nav`（改浅色，并重构为双栏） |
| 全局多处 | `#1677FF` | `--color-primary`（`#3370FF`） |
| 多页面 | `#f5f5f5` / `#fff` / `#fafafa` / `#f0f0f0` | `--bg-body` / `--bg-container` / `--bg-hover` / `--divider` |

### 9.4 落地顺序建议

1. 建 token 与 ThemeProvider，接入 AntD 亮暗 algorithm。
2. 重构 `AppLayout` 为飞书式双栏 + 浅色侧栏 + 主题切换。
3. 逐页替换 inline 色值为 token（按 9.3 清单）。
4. 重设计关键组件（KB 卡片、消息气泡、来源卡、状态徽标、KPI 卡）。
5. 逐页应用新版式（7.1–7.7）。
6. 校验亮/暗双主题、响应式断点、可读性对比度（WCAG AA）。
