# BUG-029: PERSON 正则无法识别普通中文姓名

| 属性 | 值 |
|------|---|
| **严重级别** | 🟡 Medium |
| **影响 AC** | Week 6: 实体识别准确率 |
| **发现方式** | 运行时测试 |
| **状态** | Open |

## 现象
合同中的授权代表姓名"张建国"和"李明辉"出现在正文中，但规则引擎的 PERSON 正则完全无法匹配。

## 根因
PERSON 正则仅匹配带特定前缀的姓名：
```python
"PERSON": [
    r"(?:甲方|乙方|法定代表人|签字人|负责人|联系人|法人)[：:]\s*([一-鿿]{2,4})",
],
```

这个模式要求姓名前面紧贴"甲方:"、"乙方:"等角色标注词。如果姓名出现在普通句子中（如"张建国签字"、"李明辉负责"），正则无法匹配。

## 复现
```
输入: "甲方授权代表：张建国（签字）" → ✅ 匹配 (有"甲方授权代表："前缀)
输入: "张建国签字" → ❌ 不匹配 (无角色标注前缀)
```

## 修复
增加一个纯姓名正则（2-3 字中文姓名模式），带上下文约束（跟在"代表"、"负责人"、"签字"等词前后）以避免误匹配太多普通词汇。
