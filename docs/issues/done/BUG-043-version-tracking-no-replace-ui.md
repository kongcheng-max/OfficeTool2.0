# BUG-043: BC11 · 知识库版本追踪 — 缺少文档替换入口 + 版本对比功能

| 属性 | 值 |
|------|-----|
| 发现日期 | 2026-06-28 |
| 严重程度 | **高** / High |
| 影响范围 | 文档版本管理 |
| 责任部门 | 前端 + 后端 |
| 状态 | Open |

## 现象

BC11 要求的知识库版本追踪达不到预期：
1. 不知道如何为文档创建新版本
2. 替换文档的入口完全找不到

## 根因

### 后端已就绪，前端完全缺失

后端 `api/document.py` 已实现：
- `POST /{doc_id}/replace` — 替换文档并自动创建版本快照
- `GET /{doc_id}/versions` — 版本历史列表
- `DocumentVersion` 模型已在数据库

但前端 `Documents/index.tsx` 和 `DocumentPreview.tsx` **没有任何替换文档的 UI**：
- 文档表格操作列只有「预览」和「删除」，**没有「替换」按钮**
- DocumentPreview 虽展示版本历史列表，但**没有「上传新版本」按钮**
- DocumentPreview 的版本列表展示 `v1`, `v2` 等，但没有版本之间的**差异对比**

### 缺失的功能点

| 功能 | 状态 |
|------|------|
| 版本快照存储 | ✅ 后端 |
| 版本历史 API | ✅ 后端 |
| 文档替换 API | ✅ 后端 |
| 替换文档 UI（上传新版本按钮） | ❌ 前端 |
| 版本对比 | ❌ 前后端均无 |
| 版本回退（恢复到旧版本） | ❌ 前后端均无 |

## 修复方案

### 前端（最小可行修复）

1. **文档表格操作列增加「替换」按钮**：
   ```tsx
   <Button onClick={() => setReplaceDocId(record.id)}>替换</Button>
   ```
   弹出 Modal 包含 `DocumentUpload` 组件 + 变更说明输入框

2. **DocumentPreview 版本列表增加「上传新版本」按钮**：
   在版本历史区域顶部增加 `Upload` 按钮，上传后调用 `POST /{doc_id}/replace`

### 后端（增强）

3. **版本预览/对比端点**：
   `GET /{doc_id}/versions/{version_id}/chunks` — 返回某版本的解析结果

## 影响

版本追踪是 Phase 2 核心功能，后端基建已完成但用户完全无法触发。
