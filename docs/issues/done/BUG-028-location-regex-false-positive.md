# BUG-028: LOCATION 正则误匹配公司名（"深圳市"匹配进腾讯公司名）

| 属性 | 值 |
|------|---|
| **严重级别** | 🟡 Medium |
| **影响 AC** | Week 6: 实体识别准确率 |
| **发现方式** | 运行时测试 |
| **状态** | Open |

## 现象
LOCATION 模式 `[一-鿿]{2,10}市` 将 "深圳市腾讯计算机系统有限公司" 中的 "深圳市" 误提为 LOCATION。

## 根因
LOCATION 的"市"结尾模式过于贪心，不区分后文是普通结尾还是公司名的一部分。正则匹配优先于上下文的语义判断。

## 复现
```
输入: "深圳市腾讯计算机系统有限公司"
LOCATION 匹配: "深圳市" (❌ 这是个公司的所在地前缀，不应单独提取)
ORG 匹配: "深圳市腾讯计算机系统有限公司" (✅ 正确)
```

## 修复
在 `_apply_rule_patterns` 中增加后处理：如果 LOCATION 实体是 ORG 实体的前缀子串，则丢弃 LOCATION 保留 ORG。
