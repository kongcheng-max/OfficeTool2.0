# BUG-036: 无法批量给多个文档分配标签

| 属性 | 值 |
|------|-----|
| 发现日期 | 2026-06-28 |
| 严重程度 | **中** / Medium |
| 影响范围 | 标签分配效率 |
| 责任部门 | 前端开发组 |
| 状态 | Open |

## 现象

用户不知道如何将标签「合同」分配到某个 KB、或将「2026Q2」分配到某份文档。

## 根因

### 标签分配到文档的唯一入口

`E:\OfficeTool\app\frontend\src\components\DocumentPreview.tsx:150-173`：
- 只有点击文档列表右侧的眼睛图标（预览按钮），打开 DocumentPreview 抽屉
- 在抽屉底部有一个多选下拉框列出所有 KB 标签
- 选取标签后立即调用 `assignTags` API

**这个 UI 入口极其隐蔽**，用户难以发现。

### 缺失的功能

1. **文档列表页无批量操作**：`Documents/index.tsx` 表格没有行复选框、没有「批量分配标签」按钮
2. **后端已支持批量**：`POST /api/v1/kb/{kb_id}/tags/assign` 接受 `document_ids: list[str]` + `tag_ids: list[str]`，瓶颈在前端
3. **缺少标签列**：文档列表表格没有标签列（见 BUG-037），用户看不到哪些文档已有标签

## 修复方案

1. **文档列表页增加批量操作**：
   - 表格增加行选择（checkbox column）
   - 选中行后显示「批量分配标签」按钮
   - 弹出标签选择弹窗（多选下拉 + 确认）
2. **文档列表增加标签列**：每行显示该文档已有的标签（见 BUG-037）
3. **增加入口引导**：在文档管理页面顶部或标签管理弹窗中增加说明文字

## 影响

标签系统可用性差——用户能找到标签 CRUD 但不知道如何把标签分配到文档上。
