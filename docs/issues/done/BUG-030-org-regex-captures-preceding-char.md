# BUG-030: ORG 正则捕获前置字符（"与北京字节跳动科技有限公司"）

| 属性 | 值 |
|------|---|
| **严重级别** | 🟢 Low |
| **影响 AC** | Week 6: 实体识别准确率 |
| **发现方式** | 运行时测试 |
| **状态** | Open |

## 现象
输入 "与北京字节跳动科技有限公司" 时，ORG 正则 `([一-鿿]{2,30}(?:公司|集团|...))` 把前置词"与"也捕获为实体名的一部分。

## 根因
"与" 落在 `[一-鿿]` Unicode 范围内（CJK 统一表意文字），正则贪心回溯时包含它。

## 复现
```
输入: "与北京字节跳动科技有限公司"
输出: "与北京字节跳动科技有限公司"  (❌ 多了个"与")
期望: "北京字节跳动科技有限公司"  (✅)
```

## 修复
在匹配结果上增加 strip，去除常见连词/介词前缀（"与"、"及"、"和"、"和"、"的"）。
