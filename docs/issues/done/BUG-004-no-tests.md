# BUG-004: 零测试覆盖

| 属性 | 值 |
|------|---|
| **严重级别** | 🟠 High |
| **影响 AC** | 全部 |
| **发现方式** | 代码审查 |
| **状态** | Open |

## 现象
项目中没有任何自动化测试：
- `app/tests/` 目录仅有空的 `__init__.py`
- 前端 `package.json` 中无 test 脚本或测试依赖
- `pyproject.toml` 声明了 `pytest>=8.0.0` 依赖但无测试用例

## 根因
MVP 阶段专注于功能开发，测试被推迟。

## 影响
- 无法通过自动化测试验证任何 AC
- 重构/修改风险高
- 回归 Bug 无法及时发现

## 修复建议
1. 至少为核心流程添加集成测试（注册→登录→创建KB→上传文档→问答）
2. 为解析器添加单元测试
3. 为安全模块添加单元测试
