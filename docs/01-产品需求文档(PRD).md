## 1. 产品概述

### 1.1 产品愿景

**让企业的每一份文档都成为可对话的知识。**

OfficeTool 是一套面向企业的私有化部署文档智能解析与问答系统。利用 LLM、RAG 和知识图谱技术，将海量非结构化文档转化为可检索、可问答、可关联的结构化知识资产，让员工通过自然语言快速获取文档答案。

### 1.2 产品定位

| 维度 | 描述 |
|------|------|
| 产品类型 | B2B 企业级私有部署文档智能平台 |
| 目标市场 | 中大型企业、政府机构、金融机构、律所、科研院所 |
| 核心价值 | 文档"活过来"——自然语言提问，秒级返回精准答案 |
| 差异化 | 文档解析 + RAG + 知识图谱 + 混合检索 四位一体 |
| 部署方式 | 全私有化，数据不出企业内网 |

### 1.3 产品边界

**v3.0 范围内：**
- ✅ 20+格式文档解析与索引
- ✅ RAG 自然语言问答（含答案溯源）
- ✅ 知识图谱自动构建与查询
- ✅ 多知识库隔离管理
- ✅ 混合检索（向量+BM25+图谱）+ RRF 精排
- ✅ 企业内网私有化部署

**v3.0 范围外（后续版本）：**
- ❌ 在线文档创建/编辑、实时协作
- ❌ 工作流 / BPM 引擎
- ❌ 公有云多租户 SaaS
- ❌ 移动端原生 App（仅响应式 Web）

## 2. 用户画像与场景

### 2.1 三类核心用户

| 画像 | 占比 | 身份 | 核心需求 | 频次 |
|------|------|------|---------|------|
| **A. 知识消费者** | 80% | 普通员工、业务分析师 | 用自然语言从文档中快速找到答案 | 每天数次 |
| **B. 知识管理员** | 15% | 部门文档管理员 | 批量上传、标签分类、版本管理 | 每周数次 |
| **C. IT运维** | 5% | 系统管理员 | 私有部署、安全合规、稳定运行 | 低频 |

### 2.2 核心场景

| ID | 场景 | 用户 | 优先级 |
|----|------|------|--------|
| S01 | 上传文档并提问"违约金条款是什么？"获取答案 | A | P0 |
| S02 | 在知识库中搜索"Q2销售目标"获取跨文档汇总 | A | P0 |
| S03 | 批量上传500+份历史文档自动解析索引 | B | P0 |
| S04 | 搜索人物"张三"查看关联组织/文档 | A | P1 |
| S05 | 为不同部门创建独立知识库设权限 | B | P1 |

| S07 | 在企业内网服务器 Docker 一键部署 | C | P0 |
| S08 | 配置角色权限、查看操作审计日志 | C | P2 |

## 4. 非功能需求

### 4.1 性能指标

| 指标 | 目标 |
|------|------|
| PDF 解析速度 | ≤ 10 秒/页 |
| 问答 P50 延迟 | ≤ 3 秒 |
| 问答 P95 延迟 | ≤ 10 秒 |
| 检索延迟 (Top-20) | ≤ 500ms |
| 并发问答 | ≥ 50 QPS |
| 单文件上限 | 200MB |
| 单知识库文档数 | ≤ 100,000 |

### 4.2 可用性与安全

| 维度 | 要求 |
|------|------|
| 可用率 | ≥ 99.5%（月） |
| 答案准确率 | ≥ 85%（有明确答案的问题） |
| 传输加密 | TLS 1.3 |
| 密码存储 | bcrypt + 随机盐 |
| 认证 | JWT (24h) |
| 审计 | 全量操作记录、不可删除 |

### 4.3 可扩展性

- LLM / Embedding 模型可配置切换（不绑定单一厂商）
- 向量库可替换（Milvus ↔ Qdrant）
- 解析器插件化（新增格式只需实现接口）
- 服务支持水平扩展

## 7. 风险与缓解

| 风险 | 影响 | 缓解措施 |
|------|------|---------|
| LLM API 服务中断 | 🔴 高 | 多厂商自动切换、配置热更新 |
| 文档解析质量不达标 | 🟡 中 | 多解析器组合、OCR 兜底 |
| 3个月周期 P2 来不及 | 🟡 中 | P2 按优先级排序确保核心项 |
| LLM 调用成本过高 | 🟡 中 | 缓存策略 + 检索粗排减少调用 |
| Embedding 选型不当 | 🟢 低 | 中文基准测试对比后选定 |
