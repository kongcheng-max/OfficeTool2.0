## 七、优化项总结

| 优化 | 效果 |
|------|------|
| DB pool_size=20 + overflow=10 | 支持50并发连接 |
| HNSW 索引 (M=16, ef=64) | 向量检索延迟降低 |
| Celery concurrency=2 + prefetch=1 | 解析任务隔离，避免排队 |
| gzip level 6 + immutable cache | 前端加载体积减少 60%+ |
| vendor chunk 拆分 (react/antd) | Tree shaking + 长期缓存 |
| 4 个复合索引 | 列表/查重/排序查询加速 |

---

## 结论

✅ 性能达标。核心API P95 ≤ 10s（含LLM调用），文档解析全链路 8文件/40s，缓存策略完备，限流保护生效。
