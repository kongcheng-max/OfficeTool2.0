## 测试方法

本次测试采用**全量代码审查**方式，对项目 `app/` 目录下的所有后端（~60 个 .py 文件）和前端（~25 个 .ts/.tsx 文件）源码进行逐行审查，重点检查：

- 数据一致性与完整性（前后端接口对齐、数据库操作）
- 资源管理（内存、文件、向量存储、ES 索引）
- 安全性（认证、授权、注入、速率限制）
- 错误处理（异常吞咽、用户反馈）
- 功能完整性（缺失的端点、未接入的 UI）
- 性能（事件循环阻塞、轮询浪费）

## 新发现 Bug 清单

### 🔴 Critical（阻塞发布）

| ID | 标题 | 影响 |
|----|------|------|
| **BUG-044** | KB Delete 不清理 Milvus/ES/文件资源 | 存储泄漏，长期运行后 Milvus/ES 膨胀 |
| **BUG-051** | 文档下载/预览 URL 不含 JWT token，iframe/图片/下载均 401 | 文档预览和下载功能完全不可用 |
| **BUG-054** | Alembic 迁移脚本使用 SQLite 语法，PostgreSQL 部署迁移失败 | Docker 部署时数据库初始化失败 |

### 🟠 High（重要功能缺陷）

| ID | 标题 | 影响 |
|----|------|------|
| **BUG-045** | 文档状态 'error' vs 'failed' 前后端不一致 | 失败文档筛选失效，状态显示错误 |
| **BUG-055** | 大文件上传在大小校验前已将全部数据读入内存 | 内存耗尽风险（DoS 攻击面） |

### 🟡 Medium（次要功能缺陷）

| ID | 标题 | 影响 |
|----|------|------|
| **BUG-046** | SSE 流式端点不处理客户端断开连接 | LLM API 浪费 |
| **BUG-047** | 认证端点缺少速率限制 | 暴力破解风险 |
| **BUG-052** | 知识库删除按钮未接入 UI | 用户无法通过前端删除知识库 |
| **BUG-056** | 缺少知识库更新 API 端点 | 用户无法修改知识库名称和描述 |

### 🟢 Low（体验/质量改进）

| ID | 标题 | 影响 |
|----|------|------|
| **BUG-048** | fetchUser 静默吞没错误 | Token 过期无提示，突然跳转登录页 |
| **BUG-049** | 对话历史仅存储 localStorage，多设备不同步 | 切换设备后对话丢失 |
| **BUG-050** | 密码强度无复杂度要求 | 弱密码安全风险 |
| **BUG-053** | 知识图谱画布不响应窗口缩放 | 大屏/小屏体验差 |
| **BUG-057** | 缺少实体最短路径 API 端点 | 缺失已实现但未暴露的功能 |

## 总体结论: ⚠️ 需要修复后验收

本次代码审查发现 3 个 Critical 级别 bug 必须在合并/发布前修复：

1. **BUG-044** (KB 删除资源泄漏) — 持续运行会导致存储膨胀
2. **BUG-051** (下载/预览认证缺失) — 核心功能不可用
3. **BUG-054** (Alembic PostgreSQL 不兼容) — Docker 部署失败

其他 11 个 bug 为功能增强和安全加固项，可按优先级排入后续迭代。

> 📝 本次测试报告由 QA 部门出具。研发部修复 Bug 后，QA 将进行复核，通过后移至 `docs/issues/done/`。
