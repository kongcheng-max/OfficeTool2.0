## 2. 硬件需求

### 2.1 最低配置 (开发 / POC)

| 资源 | 要求 |
|------|------|
| CPU | 8 vCPU |
| 内存 | 16 GB |
| 存储 | 100 GB SSD |
| GPU | 不需要 (使用云端 LLM API) |
| 网络 | 出网（调用 LLM/Embedding API） |

**适用场景**: 5 人以下团队 POC 验证，知识库文档 ≤ 1,000 份。

### 2.2 推荐配置 (生产环境)

| 资源 | 要求 |
|------|------|
| CPU | 16 vCPU |
| 内存 | 32 GB |
| 存储 | 500 GB SSD (RAID1) |
| GPU | 可选 (如需本地 Embedding 模型推理加速) |
| 网络 | 内网千兆 + 出网（调用 LLM API） |

**适用场景**: 50-200 用户，知识库文档 ≤ 100,000 份。

### 2.3 各组件资源消耗

| 组件 | CPU | 内存 | 存储 |

## 3. Docker Compose 编排

### 3.1 开发环境 (`docker-compose.yml`)

```yaml
version: "3.8"
services:
  postgres:
    image: postgres:15-alpine
    environment:
      POSTGRES_DB: officetool
      POSTGRES_USER: officetool
      POSTGRES_PASSWORD: ${PG_PASSWORD:-devpass}
    ports: ["5432:5432"]
    volumes: [pg_data:/var/lib/postgresql/data]

  redis:
    image: redis:7-alpine
    ports: ["6379:6379"]

  milvus:
    image: milvusdb/milvus:v2.3.0
    ports: ["19530:19530", "9091:9091"]
    environment:
      ETCD_ENDPOINTS: etcd:2379
      MINIO_ADDRESS: minio:9000
    depends_on: [etcd, minio]

  etcd:
    image: quay.io/coreos/etcd:v3.5.5
    # ...

  minio:
    image: minio/minio:latest
    ports: ["9000:9000", "9001:9001"]
    volumes: [minio_data:/data]
    command: server /data --console-address ":9001"

  elasticsearch:
    image: elasticsearch:8.11.0
    environment:
      discovery.type: single-node
      xpack.security.enabled: false
    ports: ["9200:9200"]

  neo4j:
    image: neo4j:5-community
    ports: ["7474:7474", "7687:7687"]
    environment:
      NEO4J_AUTH: neo4j/${NEO4J_PASSWORD:-devpass}
    volumes: [neo4j_data:/data]

  backend:
    build: ./backend
    ports: ["8000:8000"]
    environment:
      DATABASE_URL: postgresql+asyncpg://officetool:${PG_PASSWORD}@postgres:5432/officetool
      REDIS_URL: redis://redis:6379
      MILVUS_HOST: milvus
      LLM_PROVIDER: tongyi
      DASHSCOPE_API_KEY: ${DASHSCOPE_API_KEY}
    depends_on: [postgres, redis, milvus]

  worker:
    build: ./backend
    command: celery -A tasks worker --loglevel=info
    environment: # 同上
    depends_on: [postgres, redis, milvus]

  frontend:
    build: ./frontend
    ports: ["3000:80"]

volumes:
  pg_data: {}
  minio_data: {}
  neo4j_data: {}
```

### 3.2 生产环境调整

- 移除端口暴露（仅通过 Nginx 内网访问）
- 增加资源限制 (`deploy.resources.limits`)
- 增加健康检查 (`healthcheck`)
- ES/Neo4j 密码启用
- 日志驱动: `json-file` + `max-size: 10m`

## 4. 部署操作手册

### 4.1 一键部署

```bash
# 1. 克隆项目（从内网 Git 服务器）
git clone http://git.company.local/Officetool.git
cd OfficeTool

# 2. 配置环境变量
cp docker/.env.example docker/.env
vim docker/.env  # 填写 API Key、密码等

# 3. 启动所有服务
docker compose -f docker/docker-compose.yml up -d

# 4. 初始化数据库
docker compose exec backend alembic upgrade head

# 5. 等待服务就绪
docker compose ps  # 确认所有服务状态为 healthy

# 6. 访问
# 前端: http://<服务器IP>:3000
# API文档: http://<服务器IP>:8000/docs
```

### 4.2 环境变量清单

```bash
# === 数据库 ===
PG_PASSWORD=<强密码>

# === LLM API (国内厂商) ===
LLM_PROVIDER=tongyi          # tongyi | deepseek
DASHSCOPE_API_KEY=sk-xxx     # 通义千问 API Key
DEEPSEEK_API_KEY=sk-xxx     # DeepSeek API Key (备选)

# === Embedding ===
EMBEDDING_MODEL=shibing624/text2vec-base-chinese
# 或使用云端 API
# EMBEDDING_API_KEY=sk-xxx

# === 存储加密 (可选) ===
DOCUMENT_ENCRYPTION_KEY=<32字节随机字符串>

# === 安全 ===
JWT_SECRET_KEY=<随机生成的密钥>
JWT_ALGORITHM=HS256

# === Neo4j ===
NEO4J_PASSWORD=<强密码>
```

## 5. 配置说明

### 5.1 LLM 配置切换

```yaml
# backend/core/config.py 或 config.yaml

llm:
  provider: tongyi           # 主 LLM
  fallback: deepseek         # 备选（主 LLM 不可用时切换）
  models:
    tongyi:
      chat_model: qwen-max
      api_base: https://dashscope.aliyuncs.com/compatible-mode/v1
    deepseek:
      chat_model: deepseek-chat
      api_base: https://api.deepseek.com/v1
  timeout: 30               # 请求超时 (秒)
  max_retries: 2
```

### 5.2 模型热更新

| 配置 | 影响范围 | 更新方式 |
|------|---------|---------|
| LLM Provider | 问答生成 | 修改配置 → 重启 FastAPI |
| Embedding Model | 新文档索引 | 修改配置 → 需重建向量库 |
| 检索 Top-K | 问答精度 | 修改配置 → 即时生效 |

## 6. 数据备份与恢复

### 6.1 备份脚本

```bash
#!/bin/bash
# scripts/backup.sh

BACKUP_DIR="/data/backups/$(date +%Y%m%d_%H%M%S)"
mkdir -p "$BACKUP_DIR"

# PostgreSQL
docker compose exec -T postgres pg_dump -U officetool officetool > "$BACKUP_DIR/db.sql"

# Milvus
docker compose exec -T milvus milvus-backup ...

# Neo4j
docker compose exec -T neo4j neo4j-admin dump --to=/backups/neo4j.dump

# MinIO 文档文件
docker compose cp minio:/data "$BACKUP_DIR/minio_data"

# 打包
tar -czf "$BACKUP_DIR.tar.gz" "$BACKUP_DIR"

echo "备份完成: $BACKUP_DIR.tar.gz"
```

### 6.2 备份策略

| 类型 | 频率 | 保留 |
|------|------|------|
| 全量备份 | 每天 02:00 | 最近 7 天 |
| 增量备份 (PG WAL) | 持续 | 最近 3 天 |
| 异地备份 | 每天 | 最近 30 天 |

### 6.3 恢复流程

```bash
# 1. 停止服务
docker compose down

# 2. 恢复数据库
docker compose up -d postgres
docker compose exec -T postgres psql -U officetool officetool < backup/db.sql

# 3. 恢复文件
docker compose cp backup/minio_data minio:/data

# 4. 恢复 Neo4j
docker compose exec neo4j neo4j-admin load --from=/backups/neo4j.dump

# 5. 重建 Milvus 索引 (从 PG 元数据触发重算)
docker compose up -d
docker compose exec backend python scripts/rebuild_index.py

# 6. 验证
curl http://localhost:8000/api/v1/health
```

## 7. 监控与告警

### 7.1 健康检查端点

| 端点 | 说明 |
|------|------|
| `GET /api/v1/health` | 综合健康检查 (DB + Milvus + Redis + LLM连通性) |
| `GET /api/v1/health/ready` | Kubernetes Readiness Probe |
| `GET /api/v1/health/live` | Kubernetes Liveness Probe |

### 7.2 关键监控指标

| 维度 | 指标 | 告警阈值 |
|------|------|---------|
| API | P95 响应时间 | > 5s |
| API | 错误率 | > 5% |
| LLM | API 调用延迟 | > 10s |
| LLM | API 错误率 | > 10% |
| Celery | 任务队列长度 | > 100 |
| DB | 连接数 | > 80% max |
| 存储 | 磁盘使用率 | > 80% |
| 内存 | 各服务内存使用 | > 80% |

### 7.3 日志收集

  driver: "json-file"

## 9. 故障处理

### 9.1 常见故障

| 现象 | 可能原因 | 处理 |
|------|---------|------|
| 问答返回慢 (>30s) | LLM API 拥堵 | 检查 API 状态页；切换备用 LLM |
| 文档上传后一直"解析中" | Celery Worker 挂了 | `docker compose restart worker` |
| 搜索结果空 | Milvus 连接断开 | `docker compose restart milvus` |
| 登录失败 | PG 连接满 | 检查连接数；重启 PG |
| 问答报错 | LLM API Key 过期 | 更新 `.env` 中的 Key → 重启 |
| 前端白屏 | 前端静态文件丢失 | `docker compose up -d --build frontend` |

## 10. 安全加固清单

### 10.1 部署前检查

- [ ] 修改所有默认密码 (PG, Neo4j, MinIO, Redis)
- [ ] 生成随机 `JWT_SECRET_KEY`
- [ ] 配置 Nginx HTTPS + TLS 1.3
- [ ] 防火墙：仅开放 443 端口，其他仅 localhost
- [ ] 配置 `DOCUMENT_ENCRYPTION_KEY`（生产环境建议开启）
- [ ] API 限流启用 (`RATE_LIMIT_ENABLED=true`)
- [ ] 审计日志启用 (`AUDIT_LOG_ENABLED=true`)
- [ ] Docker 镜像来源固定（不使用 `latest` tag）
- [ ] 定期安全更新：`docker compose pull`

### 10.2 网络安全

- OfficeTool **完全内网部署**，不需要公网 IP
- 仅在调用云端 LLM/Embedding API 时需要**出网访问**
- 出网建议通过企业 HTTP 代理统一出口
- 所有内部服务通信使用 HTTP (内网，无需 HTTPS，除非合规要求)
