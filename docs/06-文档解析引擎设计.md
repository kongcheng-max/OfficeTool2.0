## 1. 设计概述

### 1.1 设计目标

将多种格式的办公文档转化为统一的 `DocumentChunk` 结构，供下游 RAG 管道消费。

| 目标 | 描述 |
|------|------|
| **格式覆盖** | 20+ 常见文档格式（分三阶段实现） |
| **可扩展** | 新增格式只需实现 `BaseParser` 接口并注册 |
| **保留结构** | 保留页码、章节、表格结构等元信息 |
| **异步处理** | 解析走 Celery 异步任务，不阻塞 API 响应 |
| **统一输出** | 所有解析器输出相同格式的 `List[Chunk]` |

### 1.2 格式覆盖计划

| 阶段 | 格式 | 数量 |
|------|------|------|
| Phase 1 (MVP) | PDF, DOCX, XLSX, TXT, MD | 5 |
| Phase 2 (Beta) | PPTX, CSV, JSON, HTML, XML + 图片(OCR) | +6 |
| Phase 3 (GA) | RTF, ODT, EPUB + 布局分析增强 | +3 |
| v3.1+ | 音视频转写 | 未来 |

## 2. 解析器架构

### 2.1 策略模式 + 注册中心

```
                          ┌────────────────────────┐
                          │     ParserRegistry      │
                          │  (单例，启动时注册)       │
                          │                         │
                          │  register(parser)       │
                          │  find_for(filename,     │
                          │          mime_type)     │
                          │  get_all()              │
                          └───────────┬─────────────┘
                                      │ 遍历
        ┌─────────┬─────────┬─────────┼─────────┬─────────┬──────┐
        │         │         │         │         │         │      │
   ┌────┴───┐ ┌───┴───┐ ┌───┴───┐ ┌───┴───┐ ┌───┴───┐ ┌───┴──┐  │
   │  PDF   │ │ DOCX  │ │ XLSX  │ │ PPTX  │ │ Text  │ │ OCR  │ ...
   └────────┘ └───────┘ └───────┘ └───────┘ └───────┘ └──────┘
        │         │         │         │         │         │
        └─────────┴─────────┴─────────┴─────────┴─────────┘
                                │
                    ┌───────────┴──────────┐
                    │   List[Chunk]        │
                    │   (统一输出结构)      │
                    └──────────────────────┘
```

### 2.2 核心接口定义

```python
from abc import ABC, abstractmethod
from dataclasses import dataclass, field
from typing import List, Optional

@dataclass
class Chunk:
    """统一的文档块输出格式"""
    content: str                           # 文本内容
    metadata: dict                         # {page, section, source, chunk_index, ...}
    chunk_type: str = "text"               # "text" | "table" | "image_caption"

class BaseParser(ABC):
    """所有解析器的抽象基类"""

    @property
    @abstractmethod
    def name(self) -> str: ...

    @property
    @abstractmethod
    def supported_extensions(self) -> List[str]: ...

    @property
    @abstractmethod
    def supported_mime_types(self) -> List[str]: ...

    @abstractmethod
    async def parse(self, file_path: str, original_filename: str) -> List[Chunk]: ...

    def supports(self, filename: str, mime_type: Optional[str] = None) -> bool:
        ext = Path(filename).suffix.lower()
        return ext in self.supported_extensions or mime_type in self.supported_mime_types
```

### 2.3 解析器注册

```python
class ParserRegistry:
    _parsers: List[BaseParser] = []

    @classmethod
    def register(cls, parser: BaseParser):
        cls._parsers.append(parser)

    @classmethod
    def find_for(cls, filename: str, mime_type: str = None) -> Optional[BaseParser]:
        for p in cls._parsers:
            if p.supports(filename, mime_type):
                return p
        return None

    @classmethod
    def get_all(cls) -> List[BaseParser]:
        return cls._parsers
```

## 3. 各格式解析方案

### 3.1 PDF 解析

| 属性 | 方案 |
|------|------|
| **库** | PyMuPDF (fitz) ≥ 1.23 |
| **阶段** | P0 / Phase 1 |
| **能力** | 逐页文字提取、表格检测、图片提取坐标 |

```python
class PDFParser(BaseParser):
    name = "pdf"
    supported_extensions = [".pdf"]
    supported_mime_types = ["application/pdf"]

    async def parse(self, file_path: str, filename: str) -> List[Chunk]:
        import fitz
        chunks = []
        doc = fitz.open(file_path)
        for page_num, page in enumerate(doc, 1):
            text = page.get_text("text")
            if text.strip():
                chunks.append(Chunk(
                    content=text.strip(),
                    metadata={"page": page_num, "source": filename},
                    chunk_type="text"
                ))
            # 表格提取
            tables = page.find_tables()
            for tb in tables:
                chunks.append(Chunk(
                    content=self._table_to_text(tb),
                    metadata={"page": page_num, "source": filename, "is_table": True},
                    chunk_type="table"
                ))
        doc.close()
        return chunks
```

**关键处理**:
- 文字型 PDF：`page.get_text("text")` 完整提取
- 扫描型 PDF（图片）：`page.get_images()` 提取图片 → OCR 兜底
- 表格：`page.find_tables()` 检测 → 提取为结构化的 Markdown 表格文本
- 大 PDF 分页流式处理，避免内存溢出

### 3.2 DOCX 解析

| 属性 | 方案 |
|------|------|
| **库** | python-docx ≥ 1.1 |
| **阶段** | P0 / Phase 1 |

```python
class DOCXParser(BaseParser):
    name = "docx"
    supported_extensions = [".docx"]
    supported_mime_types = ["application/vnd.openxmlformats-officedocument.wordprocessingml.document"]

    async def parse(self, file_path: str, filename: str) -> List[Chunk]:
        from docx import Document
        chunks = []
        doc = Document(file_path)

        for para in doc.paragraphs:
            if para.text.strip():
                # 根据样式推断章节
                section = para.style.name if para.style else "正文"
                chunks.append(Chunk(
                    content=para.text.strip(),
                    metadata={"section": section, "source": filename},
                    chunk_type="text"
                ))

        for table in doc.tables:
            text = self._table_to_markdown(table)
            chunks.append(Chunk(
                content=text,
                metadata={"source": filename, "is_table": True},
                chunk_type="table"
            ))

        return chunks
```

**关键处理**:
- 段落按文档结构顺序提取
- 保留样式信息（标题/正文/引用）作为章节标记
- 表格转 Markdown 格式，保留可读性
- 文档中的嵌入图片：导出 + OCR 处理

### 3.3 XLSX 解析

| 属性 | 方案 |
|------|------|
| **库** | openpyxl ≥ 3.1 |
| **阶段** | P0 / Phase 1 |

**策略**: 每个 Sheet → 一个 Chunk，将表格数据转换为可读的行文本：

```
[Sheet名称]
表头: 列A | 列B | 列C
行1: 值1 | 值2 | 值3
行2: 值4 | 值5 | 值6
...
```

### 3.4 PPTX 解析

| 属性 | 方案 |
|------|------|
| **库** | python-pptx ≥ 0.6 |
| **阶段** | P1 / Phase 2 |

提取每张幻灯片的标题、正文、备注。重点处理 SmartArt、表格和嵌入图片（OCR）。

### 3.5 其他格式

| 格式 | 库 / 方案 | 阶段 |
|------|----------|------|
| TXT | 标准库 `open()` | P0 |
| MD | 标准库 `open()`，保留格式标记 | P0 |
| CSV | `csv` / `pandas` | P1 |
| JSON | `json`，递归扁平化为文本 | P1 |

| ODT | `odfpy` | P3 |
| EPUB | `ebooklib` | P3 |

## 4. 统一输出标准

### 4.1 Chunk 数据结构

```python
@dataclass
class Chunk:
    content: str          # 文本内容（供 Embedding 使用）
    metadata: dict        # 元信息
    chunk_type: str       # "text" | "table" | "image_caption"
```

### 4.2 metadata 标准字段

| 字段 | 类型 | 必填 | 说明 |
|------|------|------|------|
| `source` | str | ✅ | 原始文件名 |
| `page` | int | ❌ | 页码（PDF/PPTX 有） |
| `section` | str | ❌ | 章节/标题（DOCX 有） |
| `sheet` | str | ❌ | Sheet 名称（XLSX 有） |
| `slide` | int | ❌ | 幻灯片序号（PPTX 有） |
| `is_table` | bool | ❌ | 是否为表格内容 |
| `chunk_index` | int | ❌ | 在文档内的块序号 |
| `parser_name` | str | ✅ | 解析器名称 |

### 4.3 质量要求

| 要求 | 说明 |
|------|------|
| 文本完整性 | 正文文字不丢失 |
| 表格可读性 | 表格转为 Markdown / 行文本可读格式 |
| 无乱码 | UTF-8 编码统一 |
| 过滤噪音 | 去除非内容性标记（XML标签、样式标记等） |

## 5. OCR 集成方案

### 5.1 触发场景

| 场景 | 处理 |
|------|------|
| PDF 扫描件（无文字层） | PyMuPDF 检测 text == "" 时触发 OCR |
| 图片文件 (.jpg/.png/.tiff) | 直接 OCR |
| DOCX/PPTX 嵌入图片 | 导出后 OCR |
| 单页超过 50% 面积为图片 | 整页 OCR |

### 5.2 OCR 引擎选型

| 引擎 | 优势 | 劣势 |
|------|------|------|
| **PaddleOCR** (推荐) | 中英文识别最优、表格结构识别 | 需要 PaddlePaddle 依赖 |
| Tesseract (备选) | 轻量、多语言 | 中文识别弱于 PaddleOCR |

### 5.3 OCR 处理流程

```
图片输入 → 预处理(灰度/去噪/倾斜校正)
          → PaddleOCR 文字识别
          → 文本行排序 (自上而下、从左到右)
          → Chunk(content=识别文本, chunk_type="text")
```

### 5.4 OCR 注意事项

- OCR 结果标记 `metadata.ocr: true` 供下游质量判断
- 低置信度文字标注 `metadata.ocr_confidence`
- OCR 图片存档供人工复核

## 6. 异步解析流程

### 6.1 完整流程

```
用户上传文档
      ↓
FastAPI 接收 → 存储到 MinIO → 返回 upload_id
      ↓
创建 Document DB 记录 (status=UPLOADED)
      ↓
Celery Task: parse_document(doc_id)
      ├── 1. 从 MinIO 下载文件到临时目录
      ├── 2. ParserRegistry.find_for() 匹配解析器
      ├── 3. parser.parse() → List[Chunk]
      ├── 4. 存储 Chunk 列表 (JSON) 到 MinIO
      ├── 5. 触发下游任务: embed_document(doc_id)
      └── 6. 更新 Document.status = READY
```

### 6.2 错误处理

| 场景 | 处理 |
|------|------|
| 不支持的格式 | `status=FAILED`，error="unsupported_format" |
| 解析异常 | `status=FAILED`，error=异常信息，重试 3 次 |
| 文件损坏 | `status=FAILED`，error="corrupted_file" |
| 空文档 | `status=READY`，chunk_count=0，warn="empty_document" |

## 7. 性能指标与优化

### 7.1 性能目标

| 指标 | 目标 | 度量方式 |
|------|------|---------|
| PDF 单页解析 | ≤ 10s | 含 OCR 场景 ≤ 30s |
| DOCX (10页) | ≤ 5s | |
| XLSX (1000行) | ≤ 3s | |
| 并发解析 | ≥ 10 任务并行 | Celery Worker 数 |
| 大文件 (200MB) | 正常完成不 OOM | 内存监控 |

### 7.2 优化策略

| 策略 | 说明 |
|------|------|
| 流式处理 | PDF 逐页处理，不一次加载全文档 |
| 分片上传 | 前端 resumable.js 分片上传大文件 |
| 结果缓存 | 同一文件 (MD5) 不重复解析 |
| OCR 降级 | 优先判断是否有文字层，按需触发 OCR |
| 超时控制 | 单文档解析超时 600s，超时自动标记失败 |
